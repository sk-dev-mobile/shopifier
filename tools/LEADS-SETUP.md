# Настройка сбора заявок и аналитики

Три шага, суммарно ~10 минут. После них лиды падают в Google-таблицу с пушем в Telegram, а Метрика пишет поведение и воронку.

## 1. Приёмник лидов (Google-таблица + уведомления)

1. Создай Google-таблицу (например, «Shopifier · Лиды»).
2. В ней: Расширения → Apps Script. Вставь содержимое `tools/leads-apps-script.gs` вместо пустого кода.
3. Уведомления в Telegram (опционально, но стоит того):
   - В @BotFather создай бота (`/newbot`), сохрани токен.
   - Напиши своему боту любое сообщение, затем открой `https://api.telegram.org/bot<ТОКЕН>/getUpdates` и найди `chat.id`.
   - В Apps Script: Настройки проекта (шестерёнка) → Свойства скрипта → добавь `TG_BOT_TOKEN` и `TG_CHAT_ID`.
   - Запусти функцию `testNotify` из редактора: в Telegram должно прийти тестовое уведомление.
4. Деплой: Развернуть → Новое развёртывание → тип «Веб-приложение» → выполнять от «моего имени», доступ «всем» → скопируй URL вида `https://script.google.com/macros/s/…/exec`.
5. Вставь этот URL в `docs/index.html` → `CONFIG.formEndpoint`.

## 2. Яндекс.Метрика

1. Создай счётчик на metrika.yandex.ru для `sk-dev-mobile.github.io` (Вебвизор включить).
2. Номер счётчика вставь в `docs/index.html` → `CONFIG.metrikaId` (число). Код Метрики подключится сам.
3. В Метрике создай цели типа «JavaScript-событие» с идентификаторами:
   - `form_success` — главная цель, лид
   - `form_submit`, `cta_click`, `pricing_click`, `faq_open`, `scroll_75`
4. Воронка в отчётах: визит → `scroll_75` → `cta_click` → `form_success`.

## 3. UTM-дисциплина рассылки

Каждому каналу в рассылке — персональная ссылка:

```
https://sk-dev-mobile.github.io/shopifier/?utm_source=tg&utm_campaign=batch1&utm_content=<имя_канала>
```

`utm_content` приходит вместе с лидом в таблицу: сразу видно, кто из рассылки сконвертился, даже если человек зашёл, подумал день и вернулся (метки запоминаются в браузере).

## Воронка гипотезы (ведётся в колонке «Статус» таблицы)

`новый → демо отправлено → диалог → готов платить → оплата (фикт.) / отказ`

Ориентиры для batch1 (~50 персональных сообщений): ~30 визитов, 10-12 лидов, цель — 5 доходов до оплаты. Если конверсия в лид сильно ниже 20%, меняем оффер или сегмент, а не продолжаем рассылку.
